# 運用者向けトラブルシューティングガイド

訪問看護ステーション様から「動かない」と連絡を受けたときに、原因の所在を素早く切り分け、適切な対応を取るためのガイドです。

> **重要**: 看護師さん向けの説明ではなく、**運用責任者（あなた）が判断するための内部資料**です。

---

## 🎯 まず聞くべき3つの質問

連絡を受けたら最初にこれを聞いてください：

1. **「画面に何が表示されていますか？」**（症状の特定）
2. **「いつから動かなくなりましたか？」**（変更点の特定）
3. **「最後に正常に動いたのはいつですか？」**（回帰時期の特定）

そして「**診断情報をコピー」ボタン**を押して LINE / メールで送ってもらう。

## 📊 アップデートで直せる/直せない問題の早見表

| 症状 | アップデートで直る？ | 直接的な対応 |
|------|------------------|-----------|
| 「最新版あり」表示が消えない | ✅ ほぼ確実に直る | launcher を再起動するだけ（v1.1.5 以降で自動） |
| UIが旧版のまま | ✅ 直る | 同上 |
| 起動時 Python エラー | ✅ 多分直る | 自動更新が走れば直る |
| 「ブリッジに接続できません」 | 🟡 場合による | bridge プロセス確認 |
| カード認識しない | ❌ 直らない | ハードウェア・PKCS#11 確認 |
| PIN が違うエラー | ❌ 直らない | カード設定確認・カードロック確認 |
| 「Found more than one cert」 | 🟡 信号待ち | 既知バグ → 最新版にする |
| 「FunctionFailed」 | ❌ 直らない | カードリーダー・ドライバ確認 |
| 署名後 Acrobat で「無効」 | ❌ 直らない | Acrobat 信頼設定 |
| カードリーダーが見つかりません | ❌ 直らない | USB 物理確認・ドライバ確認 |
| **launcher自身**のバグ | ❌ **インストーラ再実行が必要** | 最新 `.exe` をDL→上書きインストール |

## 🌳 症状別フローチャート

### A. 「ツールが起動しない」「デスクトップアイコンクリックしても何も起きない」

```
Q1: タスクマネージャーで bridge.py / python.exe は動いてる?
   └ Yes → ブラウザを手動で開いて http://localhost:14733/ にアクセス
   │        ├ 開ける → launcher の問題（ブラウザ起動失敗）
   │        │           └ 対応: launcher.exe 再ビルド・インストーラ再実行
   │        └ 開けない → bridge エラー
   │                    └ 対応: bridge.log の末尾を確認 → 内容で判断
   └ No  → launcher 起動失敗
            ├ launcher.log を見る (%LOCALAPPDATA%\HpkiSigner\launcher.log)
            └ よくある原因:
               • Python embed 破損 → インストーラ再実行
               • _pending/ 適用失敗 → _pending フォルダを削除して再実行
               • antivirus 誤検知 → 例外設定追加
```

### B. 「カードリーダーが見つかりません」

```
Q1: USBデバイスマネージャーで「スマートカードリーダー」が出てる?
   ├ Yes → Q2へ
   └ No  → ハードウェアまたはドライバ問題
            ├ USB を別ポートに差し替え
            ├ 別のPCで試す（ハードウェア故障判定）
            └ ベンダー公式ドライバ再インストール
                 └ ADR-MNICU2 の場合:
                   https://www.sanwa.co.jp/support/download/dl_driver_ichiran?code=ADR-MNICU2

Q2: bridge の /api/check-reader は検出する?
   ├ Yes → カード自体の問題（Q3へ）
   └ No  → Windows Smart Card サービス確認
            └ services.msc → "Smart Card" を再起動
```

### C. 「PIN を入れても通らない」

```
Q1: PIN の試行回数を確認（カード ロック直前か?）
   ├ 残り1回 → ロック避けるため停止指示
   │           └ HPKI: 発行元へ問い合わせ
   │           └ JPKI: 役所窓口へ
   └ 残り複数回 → Q2へ

Q2: PIN の種類は正しいか?
   • マイナンバーカード「署名用」: 英数字6-16桁
   • マイナンバーカード「利用者用」: 数字4桁
   • HPKI: 数字4-8桁

Q3: カードの有効期限切れではないか?
   診断情報の notAfter を確認
```

### D. 「署名は成功するが Acrobat で『無効』」

```
これはアプリの問題ではなく、Acrobat の信頼設定問題。

JPKI の場合:
   1. setup.html のステップ5「📥 Windows証明書ストアにインストール」を再実行
   2. Acrobat の「環境設定 → 署名 → 確認 → 詳細」で:
      ☑ 署名の検証時に、Windows証明書ストアでのすべてのルート証明書を信頼する
      ☑ 証明済み文書を検証時に、Windows証明書ストアでのすべてのルート証明書を信頼する
   3. Acrobat を完全終了して再起動

HPKI の場合:
   1. setup.html のステップ5 の HPKI 向け手順
   2. または、医師会・看護協会の指示に従う
```

### E. 「最新版にしたい・更新が反映されない」

```
v1.1.5 以降:
   → 単にデスクトップアイコンをダブルクリックすれば自動更新
   → 反映されない場合: _pending/.ready ファイル確認

v1.1.4 以前のバージョン:
   → 新しい hpki-signer-setup-X.Y.Z.exe を DL して実行
   (既存設定は保持される)

完全リセットしたい:
   1. アンインストール
   2. %LOCALAPPDATA%\HpkiSigner を手動削除
   3. 最新インストーラを実行
```

## 🔍 診断情報の読み方

ユーザーが送ってきた診断情報（`/api/diagnostics` の出力）の見方：

```json
{
  "version": "1.1.5",        // ← アプリのバージョン
  "os": "Windows 11",        // ← OS（Windows 10/11 のみサポート）
  "card_readers": ["ADR-MNICU2"],  // ← 検出されたリーダー（空ならハードウェア問題）
  "pkcs11_libs": [                  // ← 検出された PKCS#11 ソフト
    {"label": "HPKIカード", "type": "hpki", "path": "..."},
    {"label": "マイナンバーカード", "type": "jpki", "path": "..."}
  ],
  "log_tail": "..."          // ← 直近のログ（具体的なエラーが見える）
}
```

**チェックポイント**：
- `card_readers` が空 → カードリーダー or ドライバ問題
- `pkcs11_libs` が空 → JPKI/HPKI ソフト未インストール
- `log_tail` に `PinIncorrect` → PIN問題
- `log_tail` に `FunctionFailed` → カード or ドライバ問題
- `log_tail` に `Found more than one certificate` → 既知バグ・最新版で修正済み

## 🆘 緊急時の最終手段：完全リセット手順

すべての切り分けで原因が特定できないときの「とりあえず動くようにする」手順：

```powershell
# 1. プロセス全停止
Get-Process python -ErrorAction SilentlyContinue | Stop-Process -Force
Get-Process launcher -ErrorAction SilentlyContinue | Stop-Process -Force

# 2. 既存インストールを完全削除
Remove-Item -LiteralPath "$env:LOCALAPPDATA\HpkiSigner" -Recurse -Force
Remove-Item -LiteralPath "$env:USERPROFILE\Desktop\HPKI電子署名ツール.lnk" -ErrorAction SilentlyContinue

# 3. 最新インストーラを実行（リモート支援なら共有画面で）
# https://github.com/lifemate-inc/hpki-signer/releases/latest
```

これでもダメなら：
- ❗ カードリーダー以外のUSB機器を全部外す
- ❗ アンチウイルスを一時的に停止
- ❗ Windows のユーザーアカウントを新規作成して試す（環境問題切り分け）

ユーザーから連絡を受けた際、どこへ次にエスカレートするかの判断：

| 原因 | 次の行動 |
|------|---------|
| アプリのバグ | GitHub Issue 化 → 修正版リリース |
| ハードウェア故障 | 「リーダー買い直し」案内 |
| Windows OS の問題 | Microsoft サポート案内 |
| ベンダーソフト | J-LIS / MEDIS / サンワサプライへ案内 |
| カード自体 | 発行元（市役所、医師会、看護協会）へ案内 |
| Acrobat設定 | リモート画面共有で一緒に設定 |

ユーザーへの返信に使えるテンプレート集：

```
ご報告ありがとうございます。最新版で修正済みの可能性があります。

それでもダメな場合は、もう一度ご連絡ください。
```

### B. 「ハードウェア・ドライバの可能性」
```
カードリーダーの問題かもしれません。
1. カードリーダーを USB から一度抜き、別のポートに差し直してください
2. それでもダメな場合、PCを再起動してください
3. PC再起動後も検出できない場合、別のPCに繋いで試してください
   - 別PCで動く → 元のPCのUSBポート問題
   - 別PCでも動かない → リーダー本体の故障
```

### C. 「PIN問題（緊急）」
```
PIN を間違え続けるとカードがロックされます。
今すぐ入力を止めてください。

【マイナンバーカード署名用PIN】
- 英数字 6〜16桁
- 5回間違えるとロック → 市役所窓口で解除手続き

【HPKIカード】
- 数字 4〜8桁
- 発行元（医師会・看護協会）に再発行依頼

PIN の種類を確認しましょう。「利用者用PIN（4桁）」と
「署名用PIN（6〜16桁）」は別物です。
```

### D. 「Acrobat 信頼設定」
```

画面共有で一緒に作業しましょう。

Zoom / Teams / LINE通話、どれがよろしいですか？

設定手順:
1. Adobe Acrobat Reader を開く
2. メニュー「編集」→「環境設定」→「署名」
3. 「確認」の「詳細...」ボタンをクリック
4. 「Windows統合」セクション内の以下にチェック:
   ☑ 署名の検証時に...
   ☑ 証明済み文書を検証時に...
5. Acrobat を再起動

3分で終わります。
```

## 🛠 launcher.exe 更新が必要になるケース

`launcher.exe` は payload に含まれないため、サイレント自動更新の対象外です。
ただし v1.1.6 で大幅に堅牢化されており、年に1〜2回程度しか更新は不要な想定です。

### 「launcher 更新が必要」と判断する基準

| 変更内容 | launcher 更新必要？ |
|---------|------------------|
| bridge.py / signer.py の修正 | ❌ 不要（自動更新で配信） |
| docs/*.html の修正 | ❌ 不要 |
| 依存ライブラリのバージョンアップ（pyhanko 等） | ❌ 不要 |
| CA 証明書の追加 | ❌ 不要 |
| **launcher.json の項目を追加・削除したい** | ✅ **必要** |
| **エラーメッセージのアクション提案を追加したい** | ✅ **必要** |
| **Self-check 項目を追加したい** | ✅ **必要** |
| **新しいポート範囲のデフォルトに変更** | ✅ **必要**（既存ユーザーは launcher.json で回避可能） |
| **新しいアップデートメカニズム** | ✅ **必要** |
| **Windows のセキュリティ仕様変更対応** | ✅ **必要** |

### launcher 更新が必要なリリースの周知方法

通常リリースとは別途、**メール／LINE で全配布先に連絡**：

```
件名: 【HPKI電子署名ツール】新しいインストーラへの差し替えのお願い (v1.X.Y)

いつもご利用ありがとうございます。
今回のアップデートには launcher（起動プログラム）の修正が含まれるため、
お手数ですが新しいインストーラの実行をお願いします。

【手順（約3分）】
1. デスクトップの「電子署名ツール」が起動中なら閉じる
2. 以下からインストーラをダウンロード：
   https://github.com/lifemate-inc/hpki-signer/releases/latest
3. ダブルクリックして「次へ」を数回押すだけ
   ※ 既存の設定・PIN記録等はすべて保持されます

ご質問がありましたらお気軽にご連絡ください。
```

### バージョン確認方法（看護師さんに伝える）

```
1. デスクトップの「電子署名ツール」をダブルクリック
2. 画面の一番下に「v1.X.Y」と表示される
3. その番号を教えてください
```

### 緊急時のロールバック（最後の砦）

新しい launcher で問題が出た場合：

1. `%LOCALAPPDATA%\HpkiSigner\_backup\` に前バージョンが残っている
2. このフォルダ内のファイルをすべて親フォルダ `HpkiSigner\` にコピーで戻る
3. それでもダメな場合は GitHub Releases から旧バージョンのインストーラを取得
