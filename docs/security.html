🔒 このツールのセキュリティについて

大切な情報がどう守られているか、わかりやすく説明します。

① PIN番号は絶対にネットを通りません

一番大切なことは、あなたが入力したPIN番号は、あなたのパソコンの外に絶対に出ない ということです。

あなたのパソコンの中だけで完結 ───────────────── │ [ブラウザ] │ ↓ ① PIN入力 │ ↓ ② 自分のPCの中の「ブリッジ」へ送る │ │ [ブリッジ] (127.0.0.1) │ ↓ ③ PCに刺さったカードリーダーへ │ │ [カード（HPKI / マイナンバー）] │ │ ─────────────────────────────────────────────┘ ↑ ↑ インターネット（PINはここを通らない） ↑ [GitHub Pages] ・最初に画面（HTML）をダウンロードするだけ ・PIN や PDF は一切送られない
✅ 結論： PIN番号は「あなたのPCの中」だけで使われます。
インターネット越しに送信される瞬間はゼロです。

② CORS って何？

CORS（コルス）は 「どの Web サイトが、ご利用中のブリッジに話しかけてよいか」 を決めるルールです。

例えるなら、事業所の入口にある「受付名簿」のようなものです：

  • 名簿に名前がある人だけ、中に入って話せる
  • 知らない人がドアをノックしても、自動で追い返す

このツールでは、あなたの GitHub Pages と あなた自身のパソコン (localhost) だけが名簿に載っています。
悪意のあるWebサイト（例：怪しいメールから開いたページ）が、勝手にあなたのブリッジに話しかけて PDF を勝手に署名させようとしても、ブリッジが「あなた知らない人ですよね」と拒否します。

✅ 設定ファイル： bridge/allowed_origins.txt
ここに書かれた URL だけが受付名簿に載っています。

③ CSRFトークン って何？

たとえ「受付名簿」に名前があっても、本人かどうかを確かめる「合言葉」が必要 — それが CSRF トークンです。

仕組み：

  • ブリッジを起動するたびに、新しいランダムな合言葉が作られる
  • あなたのブラウザは、最初に合言葉を受け取る
  • 署名するときは、毎回その合言葉を一緒に送る
  • 合言葉が違うと、ブリッジは「あなたじゃないですね」と拒否

これにより、もし悪意のあるサイトが何らかの方法でブリッジに侵入しようとしても、合言葉を知らないので何もできません。

④ PINがブラウザに残らないようにする工夫

  • PIN入力欄は「パスワード型」: 画面には ●●●● と表示
  • ブラウザの自動保存・自動入力を無効化（autocomplete=off）
  • 署名ボタンを押した瞬間、入力欄を空にする
  • ブリッジに送信完了後、JavaScript変数からもPINを消す
  • タブを切り替える・閉じる際にも自動でクリア
✅ PIN番号は、必要な「ほんの一瞬」だけメモリ上にあり、用が済んだら消えます。

⑤ ログファイル (bridge.log) について

ブリッジは動作記録（ログ）を bridge/bridge.log に書き出しています。
このログには、署名証明書の情報・処理時刻・エラー内容などが記録されますが、 PIN番号は一切書かれません。

⚠️ もし不安なら： bridge/bridge.log をテキストエディタで開いて、PIN が含まれていないことを直接確認できます。

⑥ 万が一の備え：何が起きたら危ないか

このツールを安全に使うために、以下のことに気をつけてください：

  • 🚫 知らない人にPCを触らせない
    PCにログインできる人なら、bridge.log を読んだり PIN を入力できます。
  • 🚫 怪しいUSB・拡張機能をインストールしない
    ブラウザ拡張機能はPIN入力欄を盗み見れます。
  • 🚫 業務用PCで関係ないWebサイトを開かない
    悪意のあるサイトはCORS/CSRFで防いでいますが、最初から開かないのが一番安全。
  • ✅ カードリーダーから物理カードを抜く
    署名作業が終わったら、必ずカードを抜いてください。
