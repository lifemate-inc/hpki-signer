# コンプライアンス・法令対応

本ツールを医療機関へ導入される際に、IT・情報セキュリティ部門でレビューに必要な観点をまとめた資料です。

> 本書は技術仕様の観点からの解説です。
> 各事業所での法的解釈・運用判断は、顧問弁護士・所管官庁の指針に従ってください。

## 1. 医療情報3省2ガイドライン（医療情報の安全管理）

### 1.1 該当する場合の整理

本ツールが取り扱う「医療情報」の範囲によって、対応すべきガイドラインが変わります。

| ご利用形態 | 主たるガイドライン | 本ツールの該当範囲 |
|----------|------------------|------------------|
| 医療機関で医療情報を含む PDF に電子署名 | **厚労省「医療情報システムの安全管理に関するガイドライン」第 6.0 版** | 該当（電子署名処理の局面） |
| 介護事業者で利用者情報を含む PDF に電子署名 | **厚労省「医療・介護関係事業者における個人情報の適切な取扱いのためのガイダンス」** | 該当 |
| 上記のクラウドサービスとして提供 | **経産省・総務省「医療情報を取り扱う情報システム・サービスの提供事業者における安全管理ガイドライン」** | **該当しない**（本ツールはクラウドサービスではなく、ご利用者様の PC 上で完結） |

### 1.2 本ツールの位置づけ

本ツールは:

- ご利用者様の PC 内で完結する **ローカルアプリケーション** です
- 医療情報を**外部に送信しない設計**（PIN・PDF・個人情報すべて localhost 完結）
- 「医療情報を取り扱う情報システム・サービス（クラウド）」ではなく、利用者様の医療情報システムの**構成要素**としてご利用いただきます

したがって、本ツール単独でのガイドライン適合性審査の対象ではありません。
**ご利用者様の医療情報システム全体の一部として、ご利用者様のシステムが3省2ガイドラインに準拠する形で運用する必要があります**。

### 1.3 ガイドライン要件への対応マトリクス

ご利用者様の IT 部門向けに、本ツールが**直接対応している事項**を整理します。

#### 厚労省ガイドライン第 6.0 版 — 第8章「電子署名・タイムスタンプ」関連

| 要件 | 本ツールの対応 |
|------|------------|
| 真正性確保のための電子署名（公開鍵暗号方式） | ✅ HPKI / マイナンバーカードによる RSA-PKCS#1 v1.5 署名 |
| 認定認証業務の利用 | ✅ HPKI（MEDIS）・マイナンバーカード（J-LIS）はいずれも認定認証業務 |
| タイムスタンプによる時刻認証 | ✅ TSA（SSL.com / DigiCert / Sectigo 等）の RFC3161 タイムスタンプを付与 |
| 長期署名（PAdES LTV） | ✅ pyhanko による PAdES-LTV 対応。DSS（Document Security Store）に CA 証明書・CRL を埋め込み |
| 署名検証用 CA 証明書の同梱 | ✅ MEDIS HPKI CA・MHLW HPKI Root CA・J-LIS 署名用 CA 第1〜3世代を同梱 |

#### 厚労省ガイドライン第 6.0 版 — 第6章「情報システムの安全管理」関連

| 要件 | 本ツールの対応 |
|------|------------|
| 利用者の本人確認 | ✅ IC カード + PIN による 2 要素認証（所有 + 知識） |
| アクセスログ | ✅ `bridge.log` に操作時刻・署名証明書 Subject DN を記録（PIN は記録しない） |
| 通信経路の保護 | ✅ 通信は localhost (127.0.0.1) のみ。外部通信は HTTPS（GitHub/TSA/CRL） |
| 不正アクセス防止 | ✅ CORS allowlist・CSRF token・shutdown token・CSP の四重防御 |
| 改ざん防止 | ✅ アップデート時に SHA-256 ハッシュ検証（v1.1.7+） |

### 1.4 ご利用者様の運用で別途必要な対応

本ツールでは対応しない、ご利用者様の運用で対応すべき事項:

| 要件 | ご利用者様で必要な対応 |
|------|----------------------|
| PC の物理セキュリティ | カード抜き取り運用ルール・PC ロック設定 |
| カードリーダーの厳格な保管 | IC カードを業務時間外に PC から抜く |
| OS のセキュリティ更新 | Windows Update の適用 |
| マルウェア対策 | アンチウイルスソフトの導入 |
| バックアップ | 署名前 PDF のバックアップ取得 |
| ログ保管期間 | `bridge.log` の保管・定期確認（ガイドラインでは3〜5年程度を推奨） |
| 教育・周知 | ご利用者様向けの操作教育・PIN の取扱い周知 |

## 2. e-文書法（民間事業者等が行う書面の保存等における情報通信の技術の利用に関する法律）

### 2.1 e-文書法と本ツールの関係

e-文書法は、紙の書面で保存することが法律で求められている書類を、電子データで保存することを認める法律です。電子保存に求められる要件は通常:

- **見読性**: 必要な時に参照できること
- **完全性**: 改ざんされていないこと
- **機密性**: 権限のない者がアクセスできないこと
- **検索性**: 必要な書類を検索・抽出できること

### 2.2 本ツールが直接対応する要件

| 要件 | 本ツールの対応 |
|------|------------|
| 完全性（改ざん防止） | ✅ PAdES 準拠の電子署名で、署名後の改ざんを Acrobat 等で検証可能 |
| 完全性（時刻保証） | ✅ TSA タイムスタンプで「いつ署名されたか」を保証 |
| 完全性（長期検証） | ✅ PAdES LTV により、署名証明書失効後も検証可能 |
| 機密性（署名者の同定） | ✅ HPKI / マイナンバーカードの所有者 = 署名者として明確 |

### 2.3 ご利用者様で対応すべき要件

- **見読性**: PDF を開ける環境の維持（Adobe Acrobat Reader 等）
- **検索性**: ファイル名規則・フォルダ構成の運用ルール
- **保管環境**: バックアップ・アクセス制御

## 3. 電子署名法（電子署名及び認証業務に関する法律）

### 3.1 電子署名法と本ツールの関係

電子署名法では、一定の要件を満たす電子署名に手書きの署名・押印と同等の効力が認められています。

### 3.2 本ツールの電子署名の位置づけ

| カード | 認定認証業務（電子署名法上の地位） |
|--------|--------------------------------|
| HPKI カード | MEDIS（一般財団法人 医療情報システム開発センター）が**特定認証業務**として運営 |
| マイナンバーカード（署名用） | J-LIS（地方公共団体情報システム機構）が**公的個人認証法**に基づき提供 |

**いずれも電子署名法上の信頼性が確立された認証業務によるカードを利用します**。

### 3.3 留意点

- 電子署名の**法的有効性は最終的に裁判所等の判断**によります
- 本ツールは技術的に署名を施す機能を提供しますが、運用上の真正性（誰がいつ署名したか、PIN の漏洩がないか等）は**ご利用者様の運用に依存**します
- 重要書類（契約書等）での利用は、必要に応じて顧問弁護士にご相談ください

## 4. 個人情報保護法

### 4.1 本ツールにおける個人情報の取扱い

本ツールが取り扱う個人情報:

| 情報 | 取扱い |
|------|--------|
| カードの PIN | ご利用者様の PC 内 (RAM) のみ。送信なし。即時クリア |
| 署名対象 PDF の内容 | ご利用者様の PC 内のみ。送信なし |
| 署名証明書の Subject DN（氏名等） | `bridge.log` に記録（業務記録目的）。PC 内のみ |
| カードリーダー検出情報 | `bridge.log` に記録。PC 内のみ |

**配布元（lifemate-inc）には、上記のいかなる情報も送信されません**。

### 4.2 ご利用者様（個人情報取扱事業者）の義務

医療機関・介護事業者として、本ツールを利用するご利用者様は、個人情報取扱事業者として:

- 個人情報の利用目的の特定・通知
- 安全管理措置の実施
- 従業員への監督
- 漏洩時の報告義務（個人情報保護委員会への報告・本人への通知）

を遵守する必要があります。本ツールはこれらの義務の一部（安全管理措置の技術的側面）をサポートします。

## 5. 医療機器規制（薬機法）

### 5.1 本ツールは医療機器ではありません

本ツールは、医薬品・医療機器等の品質、有効性及び安全性の確保等に関する法律（薬機法）上の医療機器ではありません。

- 患者の身体に直接作用するものではない
- 診断・治療を目的とした機能を持たない
- PDF 文書への電子署名という**事務処理を支援する一般的なソフトウェア**

したがって、PMDA（医薬品医療機器総合機構）の認証・承認は不要です。

## 6. 監査ログの取扱い

### 6.1 監査ログとして利用できる項目

`bridge.log` には以下が記録されており、監査ログとして利用できます:

```
2026-05-19 10:30:00 INFO [bridge] 署名証明書 Subject: Common Name: ○○ ○○; Country: JP
2026-05-19 10:30:00 INFO [bridge] 署名証明書 SHA1: 3261CF9CCA7010DA6837A3BCAB525975BD72888E
2026-05-19 10:30:01 INFO 127.0.0.1 - - [19/May/2026 10:30:01] "POST /api/sign HTTP/1.1" 200 -
```

| 記録項目 | 監査目的での利用 |
|---------|----------------|
| 操作時刻 | 「いつ署名したか」 |
| 署名証明書 Subject DN | 「誰が署名したか」（医師名・看護師名） |
| 署名証明書 SHA1 指紋 | 「どの証明書（カード）で署名したか」 |
| API パス・ステータス | 「正常終了したか・エラーがあったか」 |

### 6.2 ログの保管

- ローテーション: 10MB × 5 世代（自動）
- 長期保管: 年に 1 回程度、`bridge.log.*` をご利用者様の安全な場所にバックアップ推奨
- 保管期間: 3省2ガイドラインで一般に推奨される 3〜5 年を目安

### 6.3 PIN 等の機密情報

以下は**ログに一切記録されません**:

- PIN（カード暗証番号）
- PDF の本文・内容
- 患者氏名・利用者氏名（PDF 内の情報）
- カードのシリアル番号
